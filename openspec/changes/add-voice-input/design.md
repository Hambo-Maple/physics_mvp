# Design: 百度语音识别集成

## Context
当前项目仅支持文字输入，需要集成语音识别功能提升交互便利性。采用百度智能云语音识别 API，通过自有后端转发保障密钥安全。

## Goals / Non-Goals

### Goals
- 实现录音启停控制，支持主流浏览器（Chrome/Firefox）
- 识别结果自动填充并发送，触发流式对话
- 非阻塞式错误提示，不使用 alert
- 视觉反馈匹配墨绿色主题
- 与现有流式输出、公式渲染功能无冲突

### Non-Goals
- 不支持实时语音识别（边说边识别）
- 不支持移动端浏览器兼容性优化
- 不实现语音唤醒功能
- 不支持多语言识别（仅中文）

## Decisions

### 1. 接口调用方式
**决策**：前端调用自有后端，后端转发百度 API
**理由**：
- 百度 API 需要 API Key 和 Secret Key，前端直连会泄露密钥
- 后端转发可以统一管理密钥，便于更换服务商
- 后端可以添加请求频率限制、日志记录等功能

**接口设计**：
```javascript
// 前端请求
POST /api/voice/recognize
Content-Type: application/json
{
  "audio": "base64编码的音频数据",
  "format": "wav"
}

// 后端响应（成功）
{
  "text": "识别的文字内容"
}

// 后端响应（失败）
{
  "error": "错误信息"
}
```

### 2. 录音实现方式
**决策**：使用浏览器原生 MediaRecorder API
**理由**：
- 原生 API，无需引入第三方库
- Chrome/Firefox 支持良好
- 可以直接获取音频 Blob，便于转换

**音频格式**：
- 优先使用 `audio/webm`（Chrome 默认）
- 降级使用 `audio/ogg`（Firefox）
- 后端统一转换为百度 API 支持的格式

### 3. 录音控制逻辑
**决策**：点击切换录音状态（开始/停止）
**状态机**：
```
idle（空闲）→ recording（录音中）→ recognizing（识别中）→ idle
```

**状态管理**：
```javascript
const voiceState = ref('idle'); // 'idle' | 'recording' | 'recognizing'
```

### 4. 识别结果处理
**决策**：识别成功后自动填充并发送
**流程**：
1. 识别成功 → 填充到 `inputValue`
2. 自动调用 `sendMessage()` 函数
3. 触发流式输出、公式渲染、可视化联动

**理由**：
- 减少用户操作步骤（不需要手动点击发送）
- 语音输入通常是完整问题，无需编辑

### 5. 错误提示方式
**决策**：使用页面顶部文字提示，3秒自动消失
**实现**：
```javascript
const showToast = (message, type) => {
  // type: 'success' | 'error'
  // 显示提示，3秒后自动消失
};
```

**样式**：
- 成功：绿色背景 (#4caf50)
- 错误：红色背景 (#f44336)
- 位置：页面顶部居中，固定定位

### 6. 视觉反馈
**决策**：按钮颜色 + 加载动画 + 文字变化
**状态样式**：
- 空闲：墨绿色 (#006644)，文字 "语音输入"
- 录音中：深红色 (#d9534f)，转圈动画，文字 "正在录音..."
- 识别中：橙色 (#f0ad4e)，转圈动画，文字 "正在识别..."

**加载动画**：
```css
@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}
```

### 7. 与流式输出的兼容
**决策**：流式输出时禁用语音按钮
**实现**：
```vue
<button :disabled="state.isGenerating || voiceState !== 'idle'">
```

**理由**：
- 避免流式输出时用户发起新的语音输入
- 保持与输入框、发送按钮的禁用逻辑一致

## Risks / Trade-offs

### 风险 1：浏览器兼容性
**风险**：Safari、IE 不支持 MediaRecorder API
**缓解**：
- 启动时检测浏览器支持情况
- 不支持时禁用语音按钮，显示提示

### 风险 2：录音权限拒绝
**风险**：用户拒绝麦克风权限
**缓解**：
- 捕获权限错误，显示友好提示
- 提示用户在浏览器设置中开启权限

### 风险 3：百度 API 识别失败
**风险**：音频质量差、网络问题、API 限额
**缓解**：
- 显示错误提示，保留输入框原有内容
- 用户可以手动编辑或重新录音

### 风险 4：音频格式兼容性
**风险**：不同浏览器录音格式不同
**缓解**：
- 后端接收多种格式（webm/ogg/wav）
- 后端统一转换为百度 API 支持的格式

### Trade-off：自动发送 vs 手动确认
**选择**：自动发送
**说明**：
- 优点：减少操作步骤，交互流畅
- 缺点：无法编辑识别结果
- 理由：语音输入通常是完整问题，识别准确率高，自动发送更符合预期

## Migration Plan
无需迁移（新增功能，不影响现有消息）
