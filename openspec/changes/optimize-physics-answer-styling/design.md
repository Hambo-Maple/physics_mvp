# Design: 物理回答排版与样式优化

## Context
当前项目已集成 KaTeX 公式渲染，但 AI 回复的排版较为简单，缺少视觉层次和重点强调。物理教学内容需要更专业的排版来提升可读性和学术感。

## Goals / Non-Goals

### Goals
- 提升物理回答的可读性和专业性
- 通过公式大小和颜色区分行内/块级公式
- 自动识别并加粗关键物理概念
- 优化间距和视觉容器，符合 A2UI 极简学术风格

### Non-Goals
- 不实现富文本编辑器
- 不支持用户自定义样式
- 不修改用户消息的样式

## Decisions

### 1. 公式样式差异化
**决策**：行内公式 18px 深灰，块级公式 22px 墨绿色
**理由**：
- 块级公式通常是核心公式，需要视觉突出
- 墨绿色与项目主题一致，增强品牌识别
- 大小差异明确区分公式类型

### 2. 关键词自动加粗
**决策**：通过关键词列表匹配，自动为物理概念加粗
**理由**：
- AI 回复格式不可控，无法要求 AI 输出 Markdown 加粗语法
- 前端自动识别更可靠，确保一致性
- 关键词列表可扩展，适应不同物理主题

**关键词列表**：
```javascript
const physicsKeywords = [
  '微分方程', '分离变量法', '初速度', '加速度', '重力加速度',
  '匀速直线运动', '自由落体', '平抛运动', '抛物线',
  '水平方向', '竖直方向', '位移', '速度', '时间',
  '牛顿第二定律', '动能', '势能', '机械能守恒'
];
```

### 3. 文本格式化顺序
**决策**：先加粗关键词，再渲染公式
**理由**：
- 避免加粗逻辑破坏公式语法（如 `\(` 被误识别）
- 确保公式内的关键词不被加粗（保持公式完整性）
- 实现方式：先用正则排除公式区域，只在纯文本区域加粗

### 4. 间距设计
**决策**：段落 16px，块级公式上下 12px，行内公式左右 4px
**理由**：
- 16px 段落间距适中，避免过于紧凑或松散
- 12px 公式间距让公式独立成块，视觉清晰
- 4px 行内公式间距避免与文字粘连

### 5. AI 消息气泡样式
**决策**：背景 #f5f5f5，内边距 20px 24px，圆角 4px
**理由**：
- 浅灰背景与白色背景区分，突出回答区域
- 20px 上下、24px 左右内边距保证内容不紧贴边缘
- 4px 圆角符合 A2UI 极简硬朗风格

## Risks / Trade-offs

### 风险 1：关键词误匹配
**风险**：关键词可能出现在非物理语境中被误加粗
**缓解**：
- 选择专业性强的物理术语
- 避免通用词汇（如"方向""时间"需谨慎）
- 未来可引入上下文分析

### 风险 2：公式内关键词被加粗
**风险**：加粗逻辑可能破坏公式语法
**缓解**：
- 先用正则提取公式区域，标记为"不可加粗"
- 只在纯文本区域应用加粗
- 测试各种公式和文本组合

### Trade-off：自动化 vs 灵活性
**选择**：自动化优先
**说明**：
- 当前阶段优先保证一致性和可维护性
- 未来如需更高灵活性，可扩展为配置化关键词列表
