# Proposal: 添加鼠标吸附与信息悬浮展示功能

## Motivation

当前平抛运动可视化画布已实现基础动画、参数调节、轨迹绘制等功能，但用户无法直观获取轨迹上任意点的详细物理信息（坐标、速度、时间等）。为提升交互体验和教学价值，需要添加「鼠标吸附 + 信息悬浮展示」功能，让用户通过鼠标悬停即可查看关键点和轨迹点的完整物理参数。

### 核心问题
1. **信息获取困难**：用户无法查看轨迹上任意点的物理参数（坐标、速度、时间等）
2. **关键点识别不明确**：初始位置、最高点、落地位置虽有标注，但缺少详细信息展示
3. **教学价值受限**：无法通过交互方式深入理解运动过程中的物理量变化

### 预期收益
1. **增强交互性**：鼠标悬停即可查看任意点的物理信息，无需额外操作
2. **提升教学价值**：关键点展示核心公式，帮助学生理解物理原理
3. **保持性能**：吸附检测仅针对已绘制点，不影响 60fps 动画流畅度

## Design Decisions

### 1. 吸附检测策略
**决策**：在 p5.js 的 `mouseMoved` 事件中实现吸附检测，优先检测关键点，再检测轨迹点。

**理由**：
- 关键点数量固定（3 个），检测成本低
- 轨迹点数量动态增长，需要优化检测范围
- 优先级机制避免冲突（关键点 > 轨迹点）

**替代方案**：
- 使用四叉树空间索引：过度设计，当前点数量（< 1000）不需要
- 全局事件监听：与 p5.js 事件系统冲突

### 2. Tooltip 渲染方式
**决策**：使用 Vue 响应式状态 + HTML DOM 渲染 tooltip，而非 p5.js canvas 绘制。

**理由**：
- KaTeX 公式渲染需要 DOM 环境
- CSS 样式控制更灵活（半透明背景、文字排版）
- 与现有 Vue 组件架构一致

**替代方案**：
- p5.js canvas 绘制：无法渲染 KaTeX 公式，样式控制受限
- 第三方 tooltip 库：增加依赖，不符合项目约束

### 3. 位置适配算法
**决策**：默认显示在鼠标右侧 10px，检测边界后自动调整为左/上/下。

**理由**：
- 右侧显示符合用户阅读习惯（从左到右）
- 边界检测避免 tooltip 超出画布
- 简单规则易于维护

**替代方案**：
- 智能定位算法（计算最优位置）：复杂度高，收益有限
- 固定位置显示：可能遮挡内容

### 4. 性能优化策略
**决策**：轨迹吸附仅检测当前已绘制的 `pathPoints` 数组，使用线性遍历。

**理由**：
- `pathPoints` 数组已限制最大 1000 个点
- 线性遍历在 1000 点规模下性能可接受（< 1ms）
- 避免引入复杂数据结构

**替代方案**：
- 空间索引（四叉树/R 树）：过度设计
- 采样检测（每 N 个点检测一次）：可能遗漏吸附点

## Risks and Mitigations

### 风险 1：Tooltip 遮挡画布内容
**影响**：中等 - 可能影响用户观察轨迹

**缓解措施**：
- 实现位置自适应算法（检测边界后调整方向）
- 半透明背景（透明度 0.9）减少遮挡感
- 鼠标移出后 0.2 秒自动消失

### 风险 2：吸附检测影响性能
**影响**：高 - 可能导致动画卡顿（< 60fps）

**缓解措施**：
- 仅检测已绘制的 `pathPoints` 数组（最大 1000 点）
- 关键点优先检测（固定 3 个点，成本极低）
- 使用 `requestAnimationFrame` 节流（避免频繁计算）

### 风险 3：与现有交互冲突
**影响**：高 - 可能破坏拖拽、点击暂停/播放功能

**缓解措施**：
- 吸附检测不修改 `isDragging` 状态
- Tooltip 显示不触发 `mousePressed` 事件
- 添加集成测试验证兼容性

### 风险 4：KaTeX 公式渲染延迟
**影响**：中等 - Tooltip 内容可能闪烁

**缓解措施**：
- 使用 `nextTick` 确保 DOM 更新后再渲染
- 缓存公式渲染结果（相同公式不重复渲染）
- 预渲染常用公式（关键点公式固定）

## Alternatives Considered

### 替代方案 1：使用第三方 tooltip 库（如 Tippy.js）
**优点**：
- 开箱即用，功能完善
- 位置自适应算法成熟

**缺点**：
- 增加外部依赖（违反项目约束）
- 样式定制受限
- 与 p5.js canvas 集成复杂

**结论**：不采用

### 替代方案 2：在 p5.js canvas 内绘制 tooltip
**优点**：
- 与现有绘制逻辑统一
- 无需额外 DOM 元素

**缺点**：
- 无法渲染 KaTeX 公式
- 文字排版控制受限
- 半透明背景实现复杂

**结论**：不采用

### 替代方案 3：点击固定 tooltip（而非悬停显示）
**优点**：
- 避免频繁显示/隐藏
- 用户可仔细阅读信息

**缺点**：
- 与现有点击暂停/播放冲突
- 需要额外关闭操作
- 交互流程变复杂

**结论**：不采用
